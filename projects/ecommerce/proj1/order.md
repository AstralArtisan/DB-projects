# **Project1：数据的捕获与构建**

实验目标：从零构建一个符合规范（3NF）的数据库，并确立分析目标，并处理真实世界的脏数据。
该实验分为3个阶段，本次完成第一阶段。

# 实验概述

**项目目标**：从零开始，确立具有实证价值的分析目标，完成数据获取与清洗，并基于规范化理论（3NF）完成数据库设计与实现。本项目重点考察对关系模型、SQL语言、完整性约束及规范化理论的综合应用。

**技术栈**：openGauss, Python, Streamlit（前端框架）

# 阶段一：领域选择与目标定义

对应章节：第1章（数据库系统概述）、第2章（关系模型）

## 1   领域选择

l 任务：选择一个感兴趣的应用领域。

l 参考领域：影视、学校课程管理、电商交易、电子竞技数据、招聘市场分析等。

l 约束：该领域的数据必须具备一定的复杂度，至少包含3个关键实体集（如电影、演员、导演）及其之间的联系。

## 2   数据源调研

l 任务：确定具体的数据来源。

l 要求：

n 数据维度要丰富（不仅有名字，还要有数值型、日期型属性，以便分析）。

n 数据规模建议大于10,000条记录。

l 示例：选定TMDB作为影视数据源。

## 3   提出分析目标

l 任务：基于你调研的数据属性，提出3个具有探索意义或实证价值的问题。这些问题将指导你后续需要抓取哪些字段，以及如何设计表结构。

l 提问范式：你的问题必须包含以下四要素：

u 【动词】+【具体指标】+【比较/条件】+【时间范围/细分领域】

l 案例示范：

u 反例（过于简单）：“搜索周星驰演了多少部电影。”（这只是检索，不是分析）

u 正例1（关联分析）：找出(动作)过去20年中(范围)，用户评分>8.0但票房回报率<1.0(条件)的电影(细分)，并统计它们主要集中在哪些题材(指标)中？

u 正例2（趋势分析）：侦测(动作)执导作品超过10部(条件)的导演(细分)，其最近3部电影的平均评分(指标)是否比其最早3部电影低20%以上(比较)？

u 正例3（比较分析）：对比(动作)同一位演员(细分)，在出演“续集电影”(条件)时的平均票房(指标)是否显著高于其出演“非续集/原创电影”？
